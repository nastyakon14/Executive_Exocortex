# Graph RAG

Источник: https://blogs.epsilonmetrics.ru/graph-rag-kak-rag-tolko-c-grafom-znanij/ 

## Что такое Graph RAG?

**Graph RAG (Graph Retrieval-Augmented Generation)** — это расширение классической архитектуры RAG, в котором на этапе извлечения информации (*Retrieval*) вместо отдельных текстовых фрагментов используются подграфы графа знаний, связанные с пользовательским запросом.

Такой подход позволяет учитывать не только текстовое сходство документов, но и связи между сущностями, что делает ответы более точными и контекстно насыщенными.

---

# Структура графа знаний

Граф знаний состоит из трёх основных компонентов.

## 1. Сущности (Entities)

Сущности представляют собой объекты, события, концепции или другие значимые элементы текста.

Примеры:

* люди;
* организации;
* продукты;
* технологии;
* события;
* места.

Каждая сущность становится **узлом (вершиной) графа**.

## 2. Отношения (Relationships)

Отношения определяют связи между сущностями.

Примеры:

* «работает в»;
* «основан в»;
* «использует»;
* «является частью».

Отношения формируют **рёбра графа**.

## 3. Сообщества (Communities)

Сообщества представляют собой группы тесно связанных сущностей и отношений.

Они выявляются с помощью алгоритмов кластеризации и помогают анализировать большие графы, разбивая их на логические тематические области.

---

# Как работает Graph RAG

Общий процесс состоит из трёх этапов:

1. **Создание графа знаний** на основе исходных данных.
2. **Поиск релевантной информации** с использованием графовых и векторных методов.
3. **Генерация ответа** на основе найденного контекста.

В отличие от классического RAG, система извлекает не отдельные документы, а связанные между собой фрагменты графа, что позволяет учитывать структуру знаний.

---

# Конвейер Graph RAG

## Этап 1. Построение графа знаний (Indexing)

На этапе индексации неструктурированные данные преобразуются в граф знаний.

### Шаг 1. Разделение текста на фрагменты

Большие документы разбиваются на небольшие логические части (обычно до 1200 токенов) с перекрытием контекста между соседними фрагментами.

Это позволяет:

* избежать потери информации;
* сохранить смысловые связи;
* повысить качество извлечения знаний.

**Пример:**

Для аналитического отчёта могут быть выделены отдельные фрагменты:

* описание метрики;
* метод расчёта;
* результаты анализа.

---

### Шаг 2. Извлечение сущностей, связей и утверждений

Каждый текстовый фрагмент анализируется языковой моделью (LLM).

Из текста извлекаются:

#### Сущности

Значимые объекты текста:

* люди;
* компании;
* продукты;
* события;
* места.

#### Связи

Отношения между сущностями:

* работает в;
* принадлежит;
* создано компанией;
* связано с событием.

#### Утверждения

Ключевые факты, выводы и знания, содержащиеся в тексте.

---

### Шаг 3. Формирование подграфов

Для каждого текстового фрагмента строится отдельный подграф.

В нём:

* вершины соответствуют сущностям;
* рёбра отражают отношения между ними.

---

### Шаг 4. Объединение подграфов

После обработки всех документов подграфы объединяются в единый граф знаний.

При объединении:

* одинаковые сущности сливаются;
* одинаковые связи объединяются;
* устраняются дубликаты.

В результате формируется единая связанная база знаний.

---

### Шаг 5. Кластеризация графа

Для поиска тематических областей граф разбивается на сообщества.

Чаще всего используются алгоритмы:

* Leiden;
* Louvain.

Сообщества отражают группы тесно связанных сущностей и отношений.

Это позволяет:

* упростить анализ графа;
* находить тематические кластеры;
* ускорять последующий поиск.

---

### Шаг 6. Создание резюме сообществ

Для каждого сообщества генерируется краткое описание.

Обычно в него входят:

* основные сущности;
* типы сущностей;
* ключевые отношения;
* важные утверждения и факты.

Такие резюме позволяют значительно ускорить поиск информации на этапе Retrieval.

---

# Этап 2. Извлечение информации (Retrieval)

После построения графа знаний система может использовать его для ответа на запросы.

Существуют два основных режима поиска.

## Глобальный поиск (Global Search)

Используется для вопросов, требующих анализа большого количества данных.

**Пример:**

> Какие технологии используются в области искусственного интеллекта?

В этом случае анализируются сразу несколько сообществ графа.

---

## Локальный поиск (Local Search)

Используется для запросов, связанных с конкретной сущностью или небольшой областью знаний.

**Пример:**

> Как работает Graph RAG?

Поиск ограничивается ближайшим контекстом соответствующей сущности.

---

# Процесс Retrieval

После получения пользовательского запроса система выполняет три основных шага.

## Шаг 1. Векторный поиск

Запрос преобразуется в эмбеддинг и используется для поиска наиболее релевантных узлов графа.

Этот этап позволяет быстро определить начальный набор сущностей, связанных с вопросом пользователя.

---

## Шаг 2. Обход графа

После определения стартовых узлов выполняется обход графа.

Система находит:

* соседние вершины;
* связанные сущности;
* дополнительные отношения;
* релевантные факты.

Благодаря этому контекст расширяется значительно сильнее, чем при использовании только векторного поиска.

---

## Шаг 3. Ранжирование результатов

Полученные данные сортируются по степени важности и релевантности.

Для этого могут использоваться алгоритмы:

* PageRank;
* Centrality Ranking;
* другие методы графового анализа.

При ранжировании учитываются:

* значимость узла в графе;
* количество связей;
* релевантность найденному запросу.

### Примечание

Ранжирование требуется не всегда.

Для небольших графов или локальных запросов этот этап может быть пропущен.

---

# Этапы Augmentation и Generation

После завершения Retrieval дальнейшая работа системы практически не отличается от классического RAG.

## Augmentation

Найденный контекст объединяется в единый набор данных для передачи языковой модели.

## Generation

LLM использует полученную информацию для генерации итогового ответа пользователю.

---

# Роль LLM в Graph RAG

Языковые модели используются на нескольких этапах работы системы.

## Построение графа знаний

LLM выполняет:

* анализ текста;
* извлечение сущностей;
* определение связей;
* выделение ключевых фактов.

## Обогащение графа

Дополнительные данные могут быть добавлены в граф на основе внешних источников знаний.

## Анализ пользовательских запросов

LLM помогает интерпретировать запрос пользователя и определить наиболее релевантные области графа.

## Генерация ответов

На основе найденного подграфа модель формирует итоговый ответ, используя:

* текстовый контекст;
* структуру графа;
* связи между сущностями.

---

# Отличия Graph RAG от классического RAG

| Классический RAG                            | Graph RAG                                           |
| ------------------------------------------- | --------------------------------------------------- |
| Извлекает документы или чанки текста        | Извлекает подграфы знаний                           |
| Основан преимущественно на векторном поиске | Использует векторный поиск и графовые связи         |
| Слабо учитывает отношения между сущностями  | Явно моделирует связи между сущностями              |
| Ограничен локальным контекстом документов   | Позволяет анализировать глобальную структуру знаний |
| Хорошо работает для поиска фактов           | Хорошо работает для сложных аналитических запросов  |

---

# Краткая схема Graph RAG

```text
Документы
    ↓
Разбиение на чанки
    ↓
LLM: сущности + связи + факты
    ↓
Подграфы
    ↓
Объединение в граф знаний
    ↓
Кластеризация сообществ
    ↓
Резюме сообществ
    ↓
Запрос пользователя
    ↓
Векторный поиск
    ↓
Обход графа
    ↓
Ранжирование
    ↓
LLM
    ↓
Ответ пользователю
```

Graph RAG объединяет преимущества векторного поиска и графов знаний, что позволяет находить не только релевантные документы, но и скрытые взаимосвязи между сущностями, обеспечивая более глубокое понимание контекста и более качественные ответы.
